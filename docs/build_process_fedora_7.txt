Build the CellML DOM API from its source code under Fedora 7
============================================================

Prerequisites
-------------

The Mozilla framework will have to be built first, if you are to use the CellML DOM API from within PCEnv.


Required packages
-----------------

- automake - 1.10-5.noarch - A GNU tool for automatically creating Makefiles
- cppunit-devel - 1.12.0-3.fc7.i386 - Libraries and headers for cppunit development
- gcc - 4.1.2-12.i386 - Various compilers (C, C++, Objective-C, java, ...)
- gcc-c++ - 4.1.2.12.i386 - C++ support for GCC
- gsl-devel - 1.8-3.fc7.i386 - Static libraries and header files for GSL development
- libxml2-devel - 2.6.29-1.fc7.i386 - Libraries, includes, etc. to develop XML and HTML applications
- python-devel - 2.5-12.fc7.i386 - The libraries and header files needed for Python development
- subversion - 1.4.3-4.i386 - Modern Version Control System designed to replace CVS


omniORB
-------

omniORB is required to build the CellML DOM API, but it has to be installed separately from the required packages above.

omniORB 4.1.0 can be downloaded from http://sourceforge.net/projects/omniorb/. You want omniORB-4.1.0.tar.gz. Then from a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/Desktop
tar –xvf omniORB-4.1.0.tar.gz
rm omniORB-4.1.0.tar.gz
cd omniORB-4.1.0
mkdir build
cd build
../configure
make
su –
cd /home/[Username]/Desktop/omniORB-4.1.0/build
make install
cd
rm –fr /home/[Username]/Desktop/omniORB-4.1.0
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Check out the source code
-------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
svn co https://svn.physiomeproject.org/svn/physiome/CellML_DOM_API/trunk/ CellML_DOM_API
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note: you will be told that the certificate is invalid, so you may want to ignore that message and accept the certificate permanently.


Build the CellML DOM API
------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/CellML_DOM_API
aclocal && autoconf && automake
./configure --enable-context --enable-annotools --enable-cuses --enable-cevas --enable-malaes --enable-ccgs --enable-cis
make
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note #1:	"aclocal && autoconf && automake" is not required if building from a source tar.
Note #2:	if the CellML DOM API is to be used within PCEnv, then the following extra parameter is to be used in the call to "configure": "--enable-xpcom=/home/[Username]/mozilla-obj/obj-i686-pc-linux-gnu/stablexr/dist".


Test the CellML API
-------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/CellML_DOM_API
make check
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
