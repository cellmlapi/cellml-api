Download and uncompress the XUL Runner SDK (only if the CellML API is to be used within OpenCell)
=================================================================================================

The latest release of the XUL Runner SDK can be downloaded from http://releases.mozilla.org/pub/mozilla.org/xulrunner/releases/, e.g. http://releases.mozilla.org/pub/mozilla.org/xulrunner/releases/1.9.0.7/sdk/xulrunner-1.9.0.7.en-US.linux-i686.sdk.tar.bz2. From there, do the following to make it available in /home/[Username]/xulrunner-sdk:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
tar -xvf xulrunner-1.9.0.7.en-US.linux-i686.sdk.tar.bz2
rm xulrunner-1.9.0.7.en-US.linux-i686.sdk.tar.bz2
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Build the CellML DOM API from its source code
=============================================

Required Linux packages
-----------------------

- automake         A tool for generating GNU Standards-compliant Makefiles
- g++              The GNU C++ compiler
- libcppunit-dev   Unit Testing Library for C++ (note: only required for testing the CellML DOM API)
- libgsl0-dev      GNU Scientific Library (GSL) â€“ development package
- libtool          Generic library support script
- libxml2-dev      Development files for the GNOME XML library
- mercurial        Scalable distributed version control system
- omniidl4         omniORB idl to C++ and Python compiler


Check out the source code
-------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
hg clone http://cellml-api.hg.sourceforge.net:8000/hgroot/cellml-api cellml
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

or the following, if you want to be able to push things to the official Mercurial repository:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
hg clone ssh://[Username]@cellml-api.hg.sourceforge.net/hgroot/cellml-api cellml
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Java (only required if you want the Java binding to the CellML DOM API)
-----------------------------------------------------------------------

Java is (obviously) required to build the Java binding to the CellML DOM API.

Java SE Development Kit 6 Update 14 can be downloaded from http://java.sun.com/javase/downloads/index.jsp. You need to click on the "Download" button under the "Java SE Development Kit" section. From there, select "Linux" as the platform, tick the "I agree to the Java SE Development Kit 6u14 License Agreement" checkbox, and click on the "Continue" button. A new page will open, giving you access to the download itself: jdk-6u14-linux-i586.bin. Once downloaded, do the following from a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~
sudo mv jdk-6u14-linux-i586.bin /usr/local/share
cd /usr/local/share
chmod +x jdk-6u14-linux-i586.bin
sudo ./jdk-6u14-linux-i586.bin
sudo rm jdk-6u14-linux-i586.bin
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Finally, add the following to /home/[Username]/.bashrc:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
export JAVA_HOME=/usr/local/share/jdk1.6.0_14
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

and then restart it:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
. ./.bashrc
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Build the CellML DOM API
------------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
aclocal && autoconf && automake
./configure --enable-context --enable-annotools --enable-cuses --enable-cevas --enable-malaes --enable-ccgs --enable-cis --enable-vacss --enable-rdf --enable-celeds
make
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Note #1: "aclocal && autoconf && automake" is not required if building from a source tar.
Note #2: if the CellML DOM API is to be used within OpenCell, then add "--enable-xpcom=/home/[Username]/xulrunner-sdk" to the "configure" command line.
Note #3: if the examples are required (e.g. CellML2C, RunCellML and ValidateCellML), then add "--enable-examples=yes" to the "configure" command line. NOTE: THIS OPTION SHOULD WORK, BUT FOR SOME REASON IS NOT RECOGNISED UNDER UBUNTU 9.04...
Note #4: if you want the Java binding to the CellML DOM API, then add "--enable-java" to the "configure" command line.


Test the CellML API
-------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
make check
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


Install the CellML API
----------------------

From a shell prompt:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
cd ~/cellml
sudo make install
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
